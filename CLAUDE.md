# Nazukware — Project Guide

## Stack
- **Backend**: Django 4.2, Gunicorn, WhiteNoise (static file serving)
- **Database**: PostgreSQL 13
- **Infrastructure**: Docker Compose, Nginx Proxy Manager
- **Server**: Raspberry Pi (ARM64), self-hosted
- **CI/CD**: GitHub Actions with self-hosted runner → auto-deploys on push to `main`

## Project Structure
```
new_server/
├── django_app/
│   ├── django_project/         # Django settings, urls.py, wsgi
│   ├── upload/
│   │   └── templates/
│   │       └── homepage_main.html   # Only active template
│   ├── mystaticfiles/          # Source static files (hero_bg1/2/3.jpg, logo.png)
│   ├── staticfiles/            # Collected output — do NOT edit, regenerated by collectstatic
│   ├── entrypoint.prod.sh      # Container startup: migrations → collectstatic → gunicorn
│   ├── Dockerfile.prod
│   └── requirements.txt
├── nginx/
├── .github/workflows/deploy.yml
├── docker-compose.yml          # Dev (runserver, DEBUG=1)
├── docker-compose.prod.yml     # Prod (gunicorn, Nginx Proxy Manager)
└── .env.dev / .env.prod        # Environment variables
```

## Active URLs
Only one route is registered in `django_project/urls.py`:
- `/` → `upload.views.homepage_main` → renders `homepage_main.html`
- `/admin/` → Django admin

All other views and apps (algo_trading, onedrive_clone, image_generator) exist but are not wired to URLs.

## Key Commands

### Production
```bash
# Deploy (also triggered automatically on git push to main)
docker compose -f docker-compose.prod.yml down
docker compose -f docker-compose.prod.yml up -d --build

# View logs
docker logs new_server-web-1 -f

# Check running containers
docker ps
```

### Development
```bash
docker compose up -d --build
```

### Django (run inside container or with venv)
```bash
python manage.py migrate
python manage.py collectstatic --noinput
python manage.py createsuperuser
```

## Static Files
- **Source**: `django_app/mystaticfiles/` — add files here
- **Output**: `django_app/staticfiles/` — auto-generated, safe to delete
- **Serving**: WhiteNoise serves from `STATIC_ROOT` via Gunicorn in prod
- **Named volume**: In prod, `static_volume` maps to `/home/app/web/staticfiles` inside the container
- After adding new static files, a full `--build` is required for prod

## Environment
- `DEBUG=1` in `.env.dev`, `DEBUG=0` in `.env.prod`
- Prod container runs as `root` (set via `user: root` in docker-compose.prod.yml) to allow cron and volume writes
- `STORAGES` in settings.py uses `whitenoise.storage.CompressedManifestStaticFilesStorage`

## CI/CD Pipeline
- GitHub Actions workflow exists at `.github/workflows/deploy.yml` but the self-hosted runner has not been set up yet
- Deploy is currently done **manually** on the server:
  ```bash
  docker compose -f docker-compose.prod.yml down
  docker compose -f docker-compose.prod.yml up -d --build
  ```

## Session Start Checklist
Before making any edits or reads at the start of a task:
1. Check the current branch (`git branch --show-current`) and status (`git status`)
2. If the working tree is **clean**:
   - If on `main`: pull latest → `git pull origin main`
   - If on a feature branch: switch to main and pull latest:
     ```bash
     git checkout main && git pull origin main
     ```
3. Create a new feature branch and proceed.

## Git Workflow
- **Never commit directly to `main`** — always create a feature branch first
- Branch naming: `features/DD_MM_YY_NN` where DD/MM/YY is today's date and NN is a zero-padded counter starting at 01
  - Example: `features/11_04_26_01`, `features/11_04_26_02`, `features/12_04_26_01`
  - Counter resets to 01 each new day
  - Check existing branches for the day with `git branch -a | grep DD_MM_YY` to find the next number
- Workflow for every change:
  1. `git checkout -b features/DD_MM_YY_NN`
  2. Make changes
  3. `git add <files>`
  4. **Ask the user for a commit message before committing** — do not auto-generate one
  5. `git commit -m "<user-provided message>"`
  6. `git push origin features/DD_MM_YY_NN`
- A PreToolUse hook enforces this — file edits will be blocked if the branch doesn't match the required format

## Frontend Dependencies (homepage_main.html)
- **Fonts**: Google Fonts — Sora (body) + Syne (headings)
- **Carousel**: Swiper.js v11 (CDN) — used for the Core Capabilities services swiper
- **No JS framework** — vanilla JS only

## Conventions
- Django template tags (`{% static %}`) must not be placed inside inline CSS `style=""` attributes — use `data-bg` attributes and apply via JavaScript instead
- `mystaticfiles/` is the only place to add static assets
- Do not add unused templates or orphaned views
- **Do not auto-create pull requests** — push the branch and stop; let the user open the PR manually
- **Always ask the user for a commit message** before running `git commit` — never auto-generate one

## Webpage Content Guidelines
- `homepage_main.html` is a **company portfolio page** for Nazukware — target audience is clients, recruiters, and potential buyers
- Write all content from a **company/studio perspective** ("we build", "our capabilities") — never from a personal/individual perspective
- Do not reference personal job titles or designations (e.g. "Tech Lead", "Senior Engineer") — these are irrelevant to the company page
- Content should be impactful and outcome-oriented, not CV-style
