## Active URLs
Only one route is registered in `django_project/urls.py`:
- `/` в†’ `upload.views.homepage_main` в†’ renders `homepage_main.html`
- `/admin/` в†’ Django admin

All other views and apps (algo_trading, onedrive_clone, image_generator) exist but are not wired to URLs.

## Static Files
- **Source**: `django_app/mystaticfiles/` вЂ” add files here
- **Output**: `django_app/staticfiles/` вЂ” auto-generated, safe to delete
- **Serving**: WhiteNoise serves from `STATIC_ROOT` via Gunicorn in prod
- **Named volume**: In prod, `static_volume` maps to `/home/app/web/staticfiles` inside the container
- After adding new static files, a full `--build` is required for prod

## Environment
- `DEBUG=1` in `.env.dev`, `DEBUG=0` in `.env.prod`
- Prod container runs as `root` (set via `user: root` in docker-compose.prod.yml) to allow cron and volume writes
- `STORAGES` in settings.py uses `whitenoise.storage.CompressedManifestStaticFilesStorage`

## CI/CD Pipeline
- GitHub Actions workflow exists at `.github/workflows/deploy.yml` but the self-hosted runner has not been set up yet
- Deploy is currently done **manually** on the server:
  ```bash
  docker compose -f docker-compose.prod.yml down
  docker compose -f docker-compose.prod.yml up -d --build
  ```

## Session Start Checklist
Before making any edits or reads at the start of a task:
1. Check the current branch: `git branch --show-current`
2. If you are **not on `main`** and there are **no uncommitted changes** (`git status` is clean), switch to main and pull latest:
   ```bash
   git checkout main
   git pull origin main
   ```
3. Then create a new feature branch and proceed.

## Git Workflow
- **Never commit directly to `main`** вЂ” always create a feature branch first
- Branch naming: `features/DD_MM_YY_NN` where DD/MM/YY is the date and NN is a zero-padded sequence number
  - Example: `features/22_03_26_00`, `features/22_03_26_01`, `features/22_03_26_02`
  - Check existing branches for the day with `git branch -a | grep DD_MM_YY` to find the next number
- Workflow for every change:
  1. `git checkout -b features/DD_MM_YY_NN`
  2. Make changes
  3. `git add <files>`
  4. **Ask the user for a commit message before committing** вЂ” do not auto-generate one
  5. `git commit -m "<user-provided message>"`
  6. `git push origin features/DD_MM_YY_NN`
- A PreToolUse hook enforces this вЂ” file edits will be blocked if the branch doesn't match the required format

## Conventions
- Django template tags (`{% static %}`) must not be placed inside inline CSS `style=""` attributes вЂ” use `data-bg` attributes and apply via JavaScript instead
- `mystaticfiles/` is the only place to add static assets
- Do not add unused templates or orphaned views
